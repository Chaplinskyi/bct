Розширення Функціоналу
Підтримка QR-кодів: 📲

Суть: Дозволити системі не тільки читати, а й генерувати та зберігати QR-коди. QR-коди можуть містити більше інформації (наприклад, термін придатності, серійний номер, посилання на документацію).

Реалізація: Потрібно буде додати бібліотеку для генерації/читання QR-кодів (наприклад, ZXing) та оновити сутності й інтерфейс для відображення/взаємодії з ними.

Мобільний Інтерфейс (PWA): 📱

Суть: Створити мобільну версію або Прогресивний Веб-Додаток (PWA), який дозволить сканувати штрих-коди камерою телефону та швидко оновлювати статус/локацію прямо зі складу.

Реалізація: Використати можливості HTML5 для доступу до камери, або розробити окремий мобільний клієнт (можливо, з використанням фреймворків типу React Native/Flutter або просто адаптивний веб-дизайн).

Розширені Ролі та Дозволи: 🔐

Суть: Замість одного "admin" (SecurityConfig.java), створити різні ролі (VIEWER, WAREHOUSE_OPERATOR, MANAGER) з різними правами (тільки перегляд, переміщення/списання, керування користувачами/імпортами).

Реалізація: Розширити SecurityConfig, можливо, додати таблиці Role та Permission у базу даних і зв'язати їх з User. Використовувати анотації Spring Security (@PreAuthorize) для контролю доступу до методів контролера.

Сповіщення про Низький Залишок: 📉

Суть: Додати можливість встановлювати мінімальний рівень залишку для певних APN. Коли загальна кількість активних штрих-кодів для APN падає нижче цього рівня, система генерує сповіщення (на дашборді, email).

Реалізація: Додати нову сутність (напр. ApnThreshold) або поле до існуючої для зберігання мінімального рівня. Створити фонове завдання (Scheduled Task) або перевірку під час списання/оновлення статусу для генерації сповіщень.

Аналітика та Звітність
Розширена Аналітика Руху: 📊

Суть: Аналізувати історію (LocationHistory, StatusHistory) для розрахунку середнього часу перебування товару на певній локації або в певному статусі. Візуалізувати "життєвий цикл" штрих-коду.

Реалізація: Додати нові JPQL/SQL запити для агрегації даних з таблиць історії. Використати Chart.js (який вже є) для нових типів графіків.

Візуалізація Складу: 🗺️

Суть: Створити інтерактивну візуальну карту складу на основі locationMap. Клікаючи на стелаж/проліт, можна було б побачити список штрих-кодів, що там знаходяться. Можна кольором показувати заповненість.

Реалізація: Використати JavaScript бібліотеку для візуалізації (наприклад, D3.js або просто динамічно генерувати HTML/CSS на основі locationMap та даних про штрих-коди).

Покращення Взаємодії
Модуль Інвентаризації: ✅

Суть: Додати спеціальний режим/сторінку для проведення інвентаризації. Користувач обирає локацію, сканує всі штрих-коди на ній. Система порівнює скановані дані з тими, що є в базі для цієї локації, і показує розбіжності (надлишки, нестачі).

Реалізація: Створити новий контролер та сторінки для процесу інвентаризації. Використовувати AJAX для надсилання сканованих кодів та отримання результатів порівняння в реальному часі.

Аудит Дій Користувачів: 📝

Суть: Записувати не тільки що змінилося (статус, локація), а й хто (який користувач) зробив цю зміну.

Реалізація: Додати поле userId або username до таблиць LocationHistory та StatusHistory. Отримувати поточного користувача з SecurityContextHolder у сервісах перед збереженням історії.
